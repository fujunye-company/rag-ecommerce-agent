# 项目结构说明

> 基于 RAG 的多模态电商 AI 导购 Agent
> 遵循开发规约 v2.0 目录规范

## 工程目录

```
02-source-code/
├── backend/                    Python FastAPI 后端
│   ├── app/
│   │   ├── main.py             FastAPI 入口 (lifespan/CORS/路由/health/version)
│   │   ├── api/                路由层 (chat/products/upload/evaluation)
│   │   ├── core/               配置 (config.py) + 数据库 (database.py)
│   │   ├── schemas/            Pydantic 数据模型 (chat/product)
│   │   ├── models/             SQLAlchemy ORM
│   │   └── services/           Agent / RAG / LLM 业务层
│   ├── alembic/                数据库迁移
│   └── requirements.txt        依赖声明
│
└── android/                    Kotlin Jetpack Compose 前端
    └── app/
        ├── build.gradle.kts    依赖配置
        └── src/main/
            ├── AndroidManifest.xml
            └── java/com/shopping/agent/
                ├── ui/         Compose 页面 + 组件
                ├── data/       Repository + API Client
                └── viewmodel/  状态管理
```

## 技术栈

| 层 | 技术 |
|----|------|
| 客户端 | Android Kotlin + Jetpack Compose |
| 后端 | Python FastAPI + SSE (sse-starlette) |
| Agent | LangGraph StateGraph |
| RAG | LlamaIndex + Qdrant |
| 向量库 | Qdrant (Docker :6333) |
| 数据库 | PostgreSQL + pgvector (Docker :5432) |
| LLM | DeepSeek v4-pro / v4-flash |

## 通信

Android → HTTP/SSE → FastAPI (:8000)
API: /health /version /api/chat /api/products /api/feedback
SSE: text_delta | product_cards | done | error
