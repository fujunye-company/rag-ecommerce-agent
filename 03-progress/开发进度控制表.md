## 依赖关系图

```
M1 工程启动 ──→ M2 数据基础 ──→ M3 RAG 检索 ──→ M4 对话闭环 ──→ M5 MVP验收
                                                      │
                                            ┌─────────┘
                                            ▼
                                    M6 多模态 ──→ M7 对比增强
                                                      │
                                                      ▼
                                              M8 评测闭环 ──→ M9 知识迭代 ──→ M10 全量交付
```

**关键依赖说明**：
- M3(RAG检索) 是最长前置链，一旦延迟会阻塞 M4/M5
- M4(SSE流式) 是前后端首次联调点，风险最高
- M5(联调) 缓冲时间必须 ≥2d

## 风险管理

| 风险点 | 影响里程碑 | 概率 | 缓解措施 |
|--------|-----------|------|---------|
| LLM API 不可用/限流 | M4/M5 | 中 | 本地 mock + fallback 回答 |
| SSE 断流不稳定 | M4/M5 | 高 | 客户端重连 + 心跳检测 |
| 商品知识数据质量问题 | M3 | 中 | 提前准备 ≥10 个标准测试商品 |
| Android 编译环境问题 | M1/M4 | 低 | 提前验证 Gradle/Maven 镜像 |
| 多模态 VLM 效果不达预期 | M6 | 高 | 图片场景降级为手动输入 |
| 时间不足 | M5+ | 高 | MVP 优先，全量内容可裁减 |

---

## 使用说明

1. **进度更新**：每个里程碑完成后在表格中打 ✅
2. **阻塞上报**：遇到 🔴 风险项立即记录阻塞原因和预计解决时间
3. **门禁检查**：每个 🚦 标记点必须通过全部 checklist 才能进入下一阶段
4. **估时跟踪**：实际耗时 vs 估时偏差 >50% 需复盘
5. **MVP 优先**：M1–M5 必须按顺序完成，M6–M9 可根据时间和资源灵活调整
