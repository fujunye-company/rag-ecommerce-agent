# 演示脚本 — AI 全栈挑战赛

> 时长：3-5 分钟 | 场景覆盖：7 级

---

## 场景 1：单轮推荐（必做）

**话术**：
> "推荐一款适合油皮的洗面奶"

**预期**：
- Agent 识别意图 → 检索商品 → 流式文本 + 商品卡片
- 卡片含图片、名称、价格、评分、匹配度

---

## 场景 2：多轮追问（必做）

**话术**：
> "帮我推荐跑鞋" → "要轻量的" → "预算500以内"

**预期**：
- Agent 保持上下文，逐步收敛需求
- 每轮更新推荐结果

---

## 场景 3：对比决策（必做）

**话术**：
> "A 和 B 哪个更保湿"

**预期**：
- Agent 提取关键维度，生成结构化对比
- 给出推荐理由

---

## 场景 4：反选/排除（加分）

**话术**：
> "推荐防晒霜，但不要含酒精的，也不要日系品牌"

**预期**：
- Agent 正确解析否定条件
- 排除命中 exclude_attributes 的商品

---

## 场景 5：场景化组合（加分）

**话术**：
> "下周三亚度假，帮我搭配一套从防晒到穿搭的方案"

**预期**：
- 跨类目检索（防晒+穿搭）
- 场景理解 + 组合编排

---

## 场景 6：购物车闭环（加分）

**话术**：
> "把刚才那款加到购物车" → "删掉第二个" → "下单吧，地址用默认的"

**预期**：
- 对话式 CRUD 操作
- 客户端实时展示购物车状态
- 模拟下单确认流程

---

## 场景 7：拍照找货（加分）

**话术**：
> [上传街拍照片] → "我想要同款"

**预期**：
- 图像理解（VLM）+ 相似商品检索

---

## 备用方案（降级）

| 场景 | 正常 | 降级 |
|------|------|------|
| LLM 超时 | Doubao API | 本地 mock 回答 |
| Qdrant 不可用 | 向量检索 | PostgreSQL 关键词搜索 |
| SSE 断流 | 流式输出 | HTTP 一次性返回 |
| 图片识别失败 | VLM | 手动输入描述词 |
