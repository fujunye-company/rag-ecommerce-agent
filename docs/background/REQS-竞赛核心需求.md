## 一、项目定义

构建**端到端电商 AI 导购 Agent**：
原生客户端（Android Kotlin） + RAG 后端 + LLM → 流式对话 + 商品卡片 + 购物车闭环。

## 三、9 级场景要求

> 来源文档实际列了 9 个场景（3 基础 + 3 进阶 + 3 高级）。  
> 评审原则：**重工程能力，重端到端完整性，关注创新深度而非广度。**

| # | 难度 | 场景 | 用户输入示例 | Agent 能力要求 | 状态 |
|:--:|:--:|------|------------|------------|:---:|
| 1 | 基础 | 单轮模糊推荐 | "推荐适合油皮的洗面奶" | 意图识别 + 属性匹配检索 | ✅ |
| 2 | 基础 | 条件筛选 | "200元以下的蓝牙耳机有哪些？" | 结构化参数提取 + 范围过滤 | ✅ |
| 3 | 进阶 | 多轮追问与细化 | "推荐跑鞋"→"轻量的"→"500以内" | 多轮上下文管理 + 渐进收敛 | ✅ |
| 4 | 进阶 | 对比决策 | "A 和 B 哪个更保湿" | 多商品并行检索 + 属性对比 + 推荐理由 | ✅ |
| 5 | 进阶 | Agent 主动反问 | "推荐一款手机"→Agent："拍照、续航还是性价比？" | 信息不足时主动澄清，引导细化需求 | ✅ |
| 6 | 高级 | 反选/排除约束 | "推荐防晒霜，不要含酒精的，不要日系品牌" | 否定条件解析 + 候选集排除 | ✅ |
| 7 | 高级 | 场景化组合推荐 | "下周三亚度假，防晒到穿搭方案" | 动态品类映射 + 品类感知MMR采样 + 场景感知生成 + 品类分组UI | ✅ |
| 8 | 高级 | 购物车与下单 | "加购"→"删第二个"→"下单，默认地址" | 对话式 CRUD + 多轮状态管理 + 业务闭环 | ✅ |
| 9 | 高级 | 拍照找货（多模态） | 上传街拍照片→"想要同款外套" | 图像理解 + 视觉特征提取 + 相似商品检索 | ✅ |

> 场景 1-5 为**必须实现**项；场景 6-9 为核心**加分项**。  
> 场景 8（购物车）是"推荐咨询→交易执行"的关键升级。  
> 场景 9（拍照找货）体现多模态 + VLM 整合能力。

## 四、加分项体系（4 大类 × 3 档）

> **评审看重深度而非广度——做精一项胜过浅尝三项。**  
> 建议在完成基础要求后选择 1-2 个方向深入实现。

### 4.1 业务闭环深度 — 购物车与下单

| 难度 | 功能点 | 说明 |
|:--:|------|------|
| ⭐ | 对话式加购 | "把这个加到购物车"，Agent 识别意图并加购，客户端展示状态 |
| ⭐⭐ | 购物车管理 | "删除第二个"、"数量改成 2" 等自然语言 CRUD |
| ⭐⭐⭐ | 下单确认流程 | Agent 引导确认地址、汇总订单，模拟下单闭环 |

> 核心考察：Agent 对结构化数据的对话式精准操作 + 客户端实时状态反馈。

### 4.2 多模态交互能力

| 难度 | 功能点 | 说明 |
|:--:|------|------|
| ⭐ | 语音输入 | 集成 ASR，语音转文字后走正常 RAG 流程 |
| ⭐⭐ | TTS 语音播报 | AI 回复支持语音朗读，打造语音导购体验 |
| ⭐⭐⭐ | 拍照找货 | 摄像头拍照 + VLM 识别物体属性 + RAG 检索相似商品 |

> 核心考察：端侧多模态采集 + 后端非文本输入→可检索语义表示。

### 4.3 对话智能与 RAG 增强

| 难度 | 功能点 | 说明 |
|:--:|------|------|
| ⭐ | 多轮上下文记忆 | 支持追问和补充条件，保持上下文连贯 |
| ⭐⭐ | 反选与排除 | 正确处理否定语义："不要含酒精的"、"除了耐克" |
| ⭐⭐⭐ | 多商品对比决策 | 自动提取关键维度，结构化对比 2-3 款商品 |

> 核心考察：Prompt 设计、检索后处理、复杂语义理解（否定/比较/条件组合）。

### 4.4 工程质量与性能优化

| 难度 | 功能点 | 说明 |
|:--:|------|------|
| ⭐ | 热门查询缓存 | 高频相似问题缓存，减少重复模型调用 |
| ⭐⭐ | 首屏极速响应 | Prompt 压缩 + 流水线并行，首 Token < 1s |
| ⭐⭐⭐ | 端侧体验打磨 | 流畅动效、骨架屏、商品卡片富交互（滑动/收藏动画） |

> 核心考察：有限资源下的工程权衡，体现生产级应用理解。

## 五、核心考察维度

### 5.1 数据工程与特征治理
- 非结构化数据向量化（商品详情/规格/评价）
- 数据一致性保障（价格/库存实时）
- Chunking 策略设计（召回率 vs 精确度平衡）

### 5.2 模型/Agent 编排
- RAG 链路可靠性 — **严禁幻觉**（编造优惠券/功能直接扣分）
- 意图识别 — 区分"随便看看"与"明确购买倾向"
- Prompt Engineering — 控制输出格式与风格
- 否定语义处理 — "不要""除了""非""不含"
- **主动追问** — 信息不足时引导用户细化需求

### 5.3 后端服务化
- SSE/WebSocket 长连接 — 低延迟流式
- 可扩展网关 — 向量检索 + LLM 调用接口
- 服务稳定性 — 并发/超时/降级

### 5.4 客户端工程
- **原生流式渲染** — 逐字流式交互（类似豆包）
- **商品卡片组件** — 对话流中嵌入可点击卡片，支持跳转
- 多模态采集 — 图片/语音端侧采集与预处理
