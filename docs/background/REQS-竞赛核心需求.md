## 一、项目定义

构建**端到端电商 AI 导购 Agent**：
原生客户端（Android Kotlin） + RAG 后端 + LLM → 流式对话 + 商品卡片 + 购物车闭环。

## 三、7 级场景要求

| 级 | 场景 | 用户输入示例 | Agent 能力 | 状态 |
|:--:|------|------------|------------|:---:|
| 1 | 单轮推荐 | "推荐适合油皮的洗面奶" | 意图识别 + 属性匹配检索 | ✅ |
| 2 | 多轮追问 | "推荐跑鞋"→"轻量的"→"500以内" | 多轮上下文 + 渐进收敛 | ✅ |
| 3 | 对比决策 | "A 和 B 哪个更保湿" | 多商品并行检索 + 属性对比 | ✅ |
| 4 | **反选/排除** | "不要含酒精的，不要日系品牌" | 否定条件解析 + 排除过滤 | ❌ |
| 5 | 场景化组合 | "下周三亚度假，防晒到穿搭方案" | 跨类目检索 + 场景理解 | ⚠️ |
| 6 | **购物车+下单** | "加购"→"删第二个"→"下单" | 对话式 CRUD + 状态管理 | ❌ |
| 7 | 拍照找货 | 上传照片→"想要同款" | 图像理解 + 相似检索 | ❌ |

> 场景 1-3 **必做**，4-7 **加分项**。场景 6（购物车）是"推荐咨询→交易执行"的关键升级。

## 四、核心考察维度

### 4.1 数据工程与特征治理
- 非结构化数据向量化（商品详情/规格/评价）
- 数据一致性保障（价格/库存实时）
- Chunking 策略设计（召回率 vs 精确度平衡）

### 4.2 模型/Agent 编排
- RAG 链路可靠性 — **严禁幻觉**（编造优惠券/功能直接扣分）
- 意图识别 — 区分"随便看看"与"明确购买倾向"
- Prompt Engineering — 控制输出格式与风格
- 否定语义处理 — "不要""除了""非"
- **主动追问** — 信息不足时引导用户细化需求

### 4.3 对话智能与 RAG 增强
- 多轮上下文记忆 — 支持追问和补充条件
- 否定语义解析 — "不要含酒精的""除了耐克还有什么"
- 多商品对比决策 — 自动提取关键维度，结构化对比

### 4.4 工程质量与性能
- 热门查询缓存 — 减少重复 LLM 调用
- **首 Token < 1s** — Prompt 压缩 + 流水线并行
- 端侧体验打磨 — 骨架屏、流畅动效、商品卡片富交互

### 4.5 后端服务化
- SSE/WebSocket 长连接 — 低延迟流式
- 可扩展网关 — 向量检索 + LLM 调用接口
- 服务稳定性 — 并发/超时/降级

### 4.6 客户端工程
- **原生流式渲染** — 逐字流式交互
- **商品卡片组件** — 对话流中嵌入可点击卡片
- 多模态采集 — 图片/语音端侧采集
