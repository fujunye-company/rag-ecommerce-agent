# 核心要求

> 电商 AI 导购系统 — 项目核心需求描述  
> 来源: 课题说明会官方文档（基于 RAG 的多模态电商智能导购 AI Agent）  
> 更新: 2026-05-21 — 同步至 Doubao 技术栈

## 系统目标

构建打通"意图理解-智能咨询-决策辅助-交易执行"全链路闭环的电商 AI 导购系统。将传统"展示型广告"升级为"交互型导购"，实现从内容浏览到购买决策的深度连接。

技术路线：Android 原生（Kotlin/Compose） ← SSE → Python FastAPI ← LangGraph → Qdrant/PostgreSQL → Doubao LLM

## 关键能力

1. **深度意图理解** — 理解商品属性、9 类用户意图（推荐/筛选/多轮/对比/反问/反选/场景/购物车/拍照），提供个性化决策支持
2. **RAG 知识库** — 上传非结构化商品详情与营销文档构建专属知识库，RAG 技术确保回复专业性与准确性
3. **流式交互体验** — 利用原生技术栈打造媲美"豆包"的流式交互，支持商品卡片实时渲染
4. **多模态输入** — 支持文字/图片/语音输入解析（拍照找货、ASR 语音转文字）
5. **评测闭环** — 构建端到端质量评测与反馈闭环，覆盖回答准确率、知识检索精度、多轮对话一致性
6. **购物车闭环** — 对话式 CRUD 操作 + 多轮状态管理 + 下单确认（加分项）

## 技术栈约束

| 层 | 要求 | 当前 |
|----|------|:---:|
| 客户端 | Android 原生框架 (Kotlin + Jetpack Compose) 或 iOS Swift | ✅ Android |
| 后端 | Python FastAPI / Gin / Express 任选 | ✅ FastAPI |
| 数据库 | PostgreSQL + pgvector | ✅ |
| 向量库 | Qdrant / Milvus / Chroma 任选 | ✅ Qdrant |

| LLM | **Doubao-Seed-2.0-lite**（比赛提供 Key） | ✅ 已验证（DeepSeek 保留降级） |
| Embedding | 任选（推荐 Doubao-embedding-vision） | ✅ BGE |

## 9 级场景要求

| # | 难度 | 场景 | Agent 能力 |
|:--:|:--:|------|-----------|
| 1 | 基础 | 单轮模糊推荐 | 意图识别 + 属性匹配检索 |
| 2 | 基础 | 条件筛选 | 结构化参数提取 + 范围过滤 |
| 3 | 进阶 | 多轮追问与细化 | 多轮上下文管理 + 渐进收敛 |
| 4 | 进阶 | 对比决策 | 多商品并行检索 + 属性对比 + 推荐理由 |
| 5 | 进阶 | Agent 主动反问 | 信息不足时主动澄清，引导细化需求 |
| 6 | 高级 | 反选/排除约束 | 否定条件解析 + 候选集排除 |
| 7 | 高级 | 场景化组合推荐 | 跨类目检索 + 场景理解 + 组合编排 |
| 8 | 高级 | 购物车与下单 | 对话式 CRUD + 多轮状态管理 + 业务闭环 |
| 9 | 高级 | 拍照找货 | 图像理解 + 视觉特征提取 + 相似检索 |

## 评分权重

| 维度 | 权重 | 评审要点 |
|------|:---:|------|
| 基础功能完整性 | 35% | 端到端链路：对话→RAG→生成→流式→商品卡片 |
| 工程质量 | 25% | 代码结构、接口设计、错误处理、文档齐全 |
| 效果与可靠性 | 20% | 流畅美观、无 Bug、检索准确、无幻觉 |
| 加分项深度 | 20% | 购物车/多模态/性能优化/否定语义 |

## 严禁项（红线）

1. 编造不存在的商品、价格、优惠信息（幻觉问题）
2. 使用纯 Web/H5 方案替代原生 App 开发
3. Demo 无法正常运行或需要大量手动配置
4. 代码完全依赖 AI 生成而无法解释原理（答辩红线）
5. API Key 泄露（GitHub 提交前放入 .gitignore）
