# 答辩演示脚本（3-5 分钟）

> 目标：在 3-5 分钟内展示 AI 导购 Agent 的端到端能力 + 技术深度
> 准备：后端运行中 + Android App 已安装 + 网络联通

---

## 开场（0:00-0:20）

**「拾物 — 基于 RAG 的多模态电商智能导购 AI Agent」**

一句话：用自然语言对话帮你找到最合适的商品，支持拍照找货、多轮对比、一键下单。

---
## 演示流程（0:20-3:30）

### 第一幕：AI 导购（0:20-1:00）

**操作**：输入「推荐适合油皮的洗面奶」

**展示点**：
- 逐字流式回复（SSE text_delta）
- 商品卡片嵌入对话流（product_cards）
- 点击卡片 → 进入商品详情页 → 查看完整信息

**口播**：「与传统搜索不同，Agent 通过 RAG 检索 + LLM 理解，用自然对话替代关键词筛选。」

### 第二幕：拍照找货（1:00-1:40）

**操作**：点击相机图标 → 拍照/选择图片 → 「想要同款外套」

**展示点**：
- VLM 图像理解 → 提取视觉特征（品类/颜色/材质/风格）
- RAG 向量检索 → 相似商品卡片
- 端侧采集 → 后端 VLM → Qdrant 检索 四层打通

**口播**：「拍照找货体现了多模态能力——VLM 提取商品属性后走 RAG 管道，不是关键词匹配而是语义理解。」

### 第三幕：多轮对话 + 反选（1:40-2:20）

**操作**：
1. 「推荐耳机」
2. 「不要骨传导的」
3. 「500以内」

**展示点**：
- 多轮上下文继承（session 状态持久化）
- 否定语义处理（must_not 排除过滤）
- Agent 主动反问（信息不足时 clarify 追问）

**口播**：「支持自然语言反选——『不要XX』『除了XX』，同时多轮上下文保持连贯，Agent 在信息不足时会主动追问。」

### 第四幕：对比决策（2:20-2:50）

**操作**：在比价页选中 2-3 件商品 → 点击「AI 对比」

**展示点**：
- 多维度结构化对比（价格/评分/品牌/属性）
- 自动标注最佳维度
- 总结推荐理由

**口播**：「AI 对比自动提取关键维度，结构化呈现差异并给出推荐理由——这是 LLM 对比传统搜索的核心优势。」

### 第五幕：购物车闭环（2:50-3:20）

**操作**：商品详情页 → 加入购物车 → 查看购物车 → 下单

**展示点**：
- 对话式加购
- 购物车数量修改/删除
- 下单确认流程（订单号生成）

**口播**：「从推荐到下单，完整业务闭环——不是只聊不买的演示，而是真正可用的购物体验。」

## 技术讲解（3:20-4:30）

### 架构亮点

**「整个系统分三层」**：

1. **客户端** — Android Kotlin 原生 Jetpack Compose，SSE 逐字流式渲染，TTS 语音播报
2. **后端** — FastAPI + LangGraph Agent 编排，10 节点工作流（classify→retrieve→rerank→generate→clarify→checkout）
3. **数据** — Qdrant 向量库 1024-dim + PostgreSQL 结构化存储，bge-large-zh-v1.5 embedding

### 防幻觉机制

**「比赛明确要求严禁幻觉——我们做了三层防护」**：

1. **RAG 检索** — 所有商品信息来自真实数据库，LLM 只做总结不做编造
2. **结构化输出** — Prompt 强制 `[PRODUCT_N]` 标记格式，卡片数据从检索 payload 填充
3. **缓存版本校验** — CACHE_VERSION 机制防止过期数据污染

### 创新点

- SSE 交错格式（摘要→商品→卡片 交替流式推送）
- 否定语义双层排除（Qdrant must_not + 文本兜底）
- VLM + RAG 拍照找货全链路
- TTS 语音导购（加分项 4.2 → 3星）
