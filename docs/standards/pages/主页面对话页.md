一、页面定位与核心说明
本页面不是独立新页面，而是主页面 HomeRecommendScreen 的「对话推荐状态」：用户在主页面输入购物需求后，Agent 通过意图理解、商品检索、RAG 知识增强和价格策略判断，在当前主页面中生成一组对话消息与商品推荐卡片。
• 页面本质：HomeRecommendScreen 内部的 ChatRecommendState，而不是新的 Activity / Screen。
• 核心任务：承接用户购物意图，将“我想买……”转化为可购买、可比价、可追问的商品推荐结果。
• 视觉目标：延续拾物项目统一的蓝粉渐变、白色圆角卡片、珊瑚红价格强调与轻量线性图标风格。
• 交互目标：保留主页面底部输入栏，允许用户继续追问、快速触发推荐、比价、论坛等功能。

四、设计想法与产品逻辑
• “每日推荐”与“对话推荐”应使用同一套 Agent 输出容器：区别在于触发源不同。每日推荐是 Agent 主动抛出，对话推荐是用户输入后触发。
• 主页面不是商品列表页，而是“导购会话页”。商品卡片、比价入口和追问气泡都服务于同一个购物决策链路。
• 推荐结果卡片要强调“为什么值得看”：通过标签、价格区间、来源平台、原价对比，把 Agent 的筛选结果具体化。
• 页面需要保留大面积留白，避免变成传统电商列表。留白代表对话空间，后续消息和推荐卡可以按时间顺序向下追加。
• 底部输入栏是主交互锚点，必须在主页面、对话态、每日推荐态中保持一致。

九、Hermes / Claude Code 构建指令
请实现“拾物 App 主页面对话态”，注意它不是新页面，而是 HomeRecommendScreen 的一个状态。

开发目标：
1. HomeRecommendScreen 需要支持三种状态：普通首页态、每日推荐态、用户对话推荐态。
2. 当前设计图对应“用户对话推荐态”：用户输入购物需求后，Agent 在主页面内返回推荐商品卡片。
3. 用户名全局统一为 fujunye，不要再出现 111111。

UI 要求：
1. 顶部保持蓝粉渐变背景，并适配前置摄像头安全区。
2. 顶部左侧为菜单图标，右侧为电话和静音图标。
3. 问候语显示为：fujunye，早上好。
4. 用户消息气泡右对齐，浅蓝白背景，最大宽度约屏幕 70%。
5. Agent 提示使用浅粉胶囊，文案为“以下是为您推荐的结果：”。
6. 推荐商品使用横向商品卡：左侧商品图，右侧商品名、标签、到手价、原价、来源、右箭头。
7. 搜索/思考状态显示为“正在搜索中... 已思考5s”，秒数由状态控制。
8. 底部输入栏必须使用统一组件：相机图标 + placeholder + 语音图标 + 加号图标。
9. 页面不跳转，新消息和商品推荐结果都追加在同一个主页面消息流中。

组件建议：
- HomeRecommendScreen
- ChatMessageList
- UserChatBubble
- AgentHintBadge
- ProductRecommendCard
- ThinkingStatusRow
- QuickActionBar
- UnifiedChatInputBar

验收标准：
1. 页面视觉风格与首页、比价页、探索页、我的页统一。
2. 推荐对话态不是独立路由，而是 HomeRecommendScreen 内部状态。
3. 商品推荐卡支持从 Mock 数据读取。
4. 输入栏可继续发送追问。
5. ThinkingStatusRow 可根据 isSearching 和 thinkingSeconds 动态显示。
6. 点击商品卡可进入商品详情或比价详情。
