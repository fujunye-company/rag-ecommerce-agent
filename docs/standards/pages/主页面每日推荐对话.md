一、设计定位：每日推荐是 Agent 主动对话，不是独立页面
• 该界面本质上属于主页面 HomeRecommendScreen 的一种对话状态：用户进入 App 后，Agent 根据用户画像、历史搜索、收藏、购物车与价格偏好，主动抛出“以下是今日推荐”的推荐对话。
• 因此前端不应新建 DailyRecommendPage 或 DailyRecommendScreen，而应在首页的 ChatShell / HomeRecommendScreen 内渲染一个 AgentInitiatedRecommendationState。
• 每日推荐区域要被设计成“系统主动发起的一轮导购对话”：有问候语、有推荐提示胶囊、有商品卡片、有输入栏，用户可继续追问、切换比价、进入探索或查看商品详情。

七、Hermes / Claude Code 构建指令
请重构主页面的“每日推荐”模块。注意：每日推荐不是新的页面，而是 HomeRecommendScreen 内由 Agent 主动抛出的一条推荐对话状态。

开发要求：
1. 页面仍属于 HomeRecommendScreen，不要新建 DailyRecommendScreen。
2. 用户名统一为 fujunye。
3. 页面背景使用项目统一蓝粉渐变 GradientScreenBackground。
4. 顶部保留菜单、电话、静音图标。
5. 问候语区域显示 “fujunye，早上好”。
6. “以下是今日推荐：”需要作为 AgentRecommendBadge 渲染，表示 Agent 主动推荐。
7. 推荐商品区域使用 DailyRecommendPanel，内部 2×2 ProductCardVertical。
8. 商品卡片包含：图片、标题、标签、到手价、原价、来源、右箭头。
9. 每日推荐数据必须来自 mock 或接口，不能硬编码在组件内部。
10. 底部输入栏使用 UnifiedChatInputBar，包含相机、文本输入、语音、加号。
11. 底部导航为：首页、比价、探索、我的，首页选中态为红色。
12. 用户可在输入栏继续追问，后续消息进入 ChatShell 消息流。

组件建议：
- HomeRecommendScreen
- GradientScreenBackground
- HomeTopActionBar
- GreetingTitle
- AgentRecommendBadge
- DailyRecommendPanel
- ProductCardVertical
- UnifiedChatInputBar
- MainBottomNavBar

验收标准：
1. 每日推荐被实现为首页对话状态，而不是独立页面。
2. 推荐内容可以由 mock 数据驱动。
3. UI 风格与比价页、探索页、我的页一致。
4. 四个商品卡片排版稳定，不重叠、不溢出。
5. 输入栏可继续触发对话。
6. 核心组件提供 Preview。
