一、页面理解与设计定位
该页面不是独立的新一级页面，而是“比价页”的展开态 / 详情态。用户在比价页点击某一商品卡片后，页面下方弹出一个白色圆角内容面板，面板中展示该商品不同颜色、不同套装、不同平台来源的具体价格跟踪信息。
设计上，它像一幅从下方拉起的“挂画式价格跟踪表”：顶部保留比价页的搜索栏、分类栏和上方两个主商品卡；中间通过一个灰色下拉滑条作为面板控制柄；下方展示可滚动的具体跟踪表。用户向下拖动滑条即可收起面板，回到普通比价页面。
• 页面本质：ComparePriceScreen 的 expanded detail state，不是新 Tab。
• 核心目标：把“某一商品的不同颜色 / 套装价格趋势”集中展示，帮助用户判断哪一个版本更值得买。
• 交互入口：点击比价页顶部商品卡、或点击任一价格浮动卡片。
• 交互出口：拖动顶部灰色下拉滑条向下收起，恢复到比价页基础状态。

四、挂画式跟踪表设计细节
“挂画式跟踪表”是本页面的核心差异点。它不应该像普通商品列表一样平铺，而应该像一张被用户从下方拉起的价格分析板。视觉上需要有面板边界、顶部控制柄和统一卡片节奏。
• 面板顶部：使用大圆角，左右贴近屏幕边缘但保留 0-8dp 安全空隙。
• 控制柄：灰色短条，宽约 48dp，高约 5dp，居中；支持向下拖动。
• 面板内容：列表中的每张卡片大小一致，高度统一，纵向滚动。
• 卡片左侧：商品变体图，白底，统一裁切比例。
• 卡片右侧：价格曲线图，用红色线条和浅粉渐变填充。
• 卡片底部：左侧显示最低价来源，右侧显示最低价，价格用红色加粗。

八、Hermes / Claude Code 开发术语转化
以下指令可直接给 Hermes 或 Claude Code 使用，用于把当前设计转化为 Android Compose 页面。
请在 ComparePriceScreen 中实现“比价详情挂画面板”。该面板不是独立页面，而是比价页的 expanded detail state。
核心要求：
1. 用户点击顶部商品卡或某个价格趋势卡片后，展开 CompareDetailSheet。
2. CompareDetailSheet 从屏幕下方弹出，视觉上像一幅挂画式价格跟踪表。
3. 面板顶部有 CompareDetailPullHandle，灰色圆角短条，居中显示。
4. 用户向下拖动 PullHandle，面板收起，回到比价页基础状态。
5. 面板中展示 VariantPriceTrackList，每个 VariantPriceTrackCard 代表同一商品的不同颜色、套装或平台版本。
6. 每个 VariantPriceTrackCard 左侧是商品图，右侧是 PriceTrendChart。
7. PriceTrendChart 使用红色价格曲线，浅粉渐变面积背景，横轴显示 30天前、15天前、今天。
8. 每个卡片底部显示 LowestPriceInfoRow：左侧为“最低价来源：平台名”，右侧为“最低价 ¥xxx”。
9. 页面整体继续使用蓝粉渐变背景、白色圆角卡片、红色价格强调，与首页、探索页、我的页保持一致。
10. 底部导航保留：首页、比价、探索、我的；当前比价 Tab 高亮。
推荐组件：
- ComparePriceScreen
- CompareDetailSheet
- CompareDetailPullHandle
- VariantPriceTrackList
- VariantPriceTrackCard
- PriceTrendChart
- LowestPriceInfoRow
验收标准：
1. 点击商品后能展开详情面板。
2. 下拉滑条能收起面板。
3. 同款商品不同颜色 / 套装能以列表方式展示。
4. 每个卡片都有商品图、价格曲线、最低价来源、最低价。
5. 价格曲线能从 Mock 数组绘制，不允许只放静态图片。
6. UI 风格必须与现有拾物 App 保持一致。
九、验收标准
验收项
标准
入口行为
点击商品卡后能展开详情挂画面板。
返回行为
下拉滑条后能收起详情面板并回到比价页。
信息完整性
每个变体卡必须包含商品图、价格曲线、最低价来源、最低价。
图表真实性
价格曲线由 mock priceTrend 数组绘制，不能用静态截图代替。
视觉统一
蓝粉渐变、白色圆角、红色价格、线性图标与全项目一致。
适配性
在 1080×2400 和 720×1600 设备上无文字溢出、无卡片重叠。
组件化
CompareDetailSheet、VariantPriceTrackCard、PriceTrendChart 必须可复用。

• P1：完成面板展开 / 收起动画与下拉手势阈值。
