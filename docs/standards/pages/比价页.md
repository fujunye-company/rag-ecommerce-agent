ComparePriceScreen Design Specification & Hermes Build Brief
文档用途：用于前端开发、Hermes / Claude Code 构建、UI 素材整理、Mock 数据设计与验收检查。
页面定位：比价页是“拾物”的价格决策核心页面，负责将同款商品、不同颜色、不同套装或不同来源的价格变化以卡片化方式呈现，帮助用户判断当前是否值得购买。
图 1  比价页面参考 UI：顶部商品双卡 + 下拉滑条 + 同款价格趋势列表
一、页面设计想法
比价页不是普通商品列表，而是“价格判断页面”。页面上方先用两张大商品卡吸引用户选择一个目标商品；用户点击某个商品后，页面下方展开同款商品的颜色、套装、来源平台的价格趋势卡片。整体交互类似一个可下拉的价格跟踪面板：上方保持商品浏览入口，下方白色面板像挂画一样展示连续价格曲线。
页面风格延续项目统一视觉：蓝粉柔和渐变背景、白色圆角卡片、轻阴影、红色价格强调、线性图标、顶部挖孔状态栏、安全区适配。
二、完整页面结构
比价页由“页面背景层、顶部搜索与分类层、主商品卡片层、价格跟踪挂画面板、底部导航层”五部分组成。

九、Hermes / Claude Code 构建指令
以下内容可直接发给 Hermes 或 Claude Code，用于开发比价页。
请实现“拾物”App 的 ComparePriceScreen，比价页要求如下：

1. 页面定位
- 比价页用于展示同款商品的价格趋势和不同颜色/套装的最低价来源。
- 点击顶部某个商品卡后，在页面中部展开挂画式价格跟踪面板。
- 该面板不是独立页面，而是 ComparePriceScreen 的 expanded state。

2. 统一视觉
- 使用项目统一蓝粉渐变背景。
- 搜索栏、分类栏、商品卡、底部导航与现有首页/探索页/我的页保持一致。
- 价格统一使用 BrandRed。
- 卡片统一白底、大圆角、轻阴影。

3. 顶部区域
- 使用 UnifiedSearchBar：左侧相机图标，中间 placeholder 为“搜索商品名称或粘贴链接”，右侧红色搜索按钮。
- 分类栏包含：推荐、零食、男装、运动、数码家电、分类。
- 当前默认选中“运动”。

4. 顶部商品卡
- 使用两列 ProductCardVertical。
- 商品卡包含商品图、标题、标签、价格、付款人数。
- 点击商品卡后设置 selectedProductId，并展开 CompareTrackingSheet。

5. 挂画式价格跟踪面板
- 使用 CompareTrackingSheet 组件。
- 面板顶部有灰色短横条 CompareSheetDragHandle。
- 用户向下拖动 handle 后收起面板，回到普通比价页。
- 面板中展示 ProductTrendCard 列表。

6. ProductTrendCard
- 左侧展示商品变体图片。
- 右侧展示 PriceTrendChart。
- 图表包含红色价格折线、浅粉面积背景、今天空心圆点。
- 横轴展示：30天前、15天前、今天。
- 纵轴展示：¥240、¥320、¥400 等。
- 底部左侧展示：最低价来源：平台名。
- 底部右侧展示：最低价 ¥xxx。

7. 数据
- 使用 assets/mock/compare.mock.json。
- topProducts 渲染顶部商品卡。
- compareVariants 根据 selectedProductId 渲染价格趋势卡。

8. 验收标准
- 点击顶部商品卡后能展开价格跟踪面板。
- 向下拖动灰色滑条能收起面板。
- 价格曲线能根据 mock 的 trend 数组绘制。
- 页面底部导航当前选中“比价”。
- 页面整体视觉与项目已有 UI 保持一致。

设计目标 | 说明
价格透明 | 通过 30 天价格曲线、最低价来源、当前最低价帮助用户判断是否值得买。
同款聚合 | 点击某一商品后展示该商品不同颜色、不同套装、不同来源平台的价格浮动。
低成本理解 | 用户不用进入复杂详情页，在比价页内即可看懂价格趋势。
保持统一 | 搜索栏、分类栏、底部导航、卡片圆角和价格色与首页、探索页、我的页保持一致。

交互行为 | 触发方式 | 结果 | 开发说明
点击顶部商品卡 | 点击黄色/白色商品卡任意区域 | 展开该商品的价格跟踪挂画面板 | 设置 selectedProductId，加载 compareVariants。
下拉滑动条 | 拖动灰色 handle 向下 | 收起价格跟踪挂画面板，回到普通比价页面 | 使用 BottomSheetScaffold 或自定义 draggable sheet。
点击趋势卡片 | 点击某个颜色/套装价格卡 | 进入该变体商品详情或展开更细价格记录 | 可先预留 onVariantClick。
点击价格曲线 | 点击曲线区域 | 查看更长周期价格详情 | 可弹出 7/30/90 天切换，MVP 先不做。
点击分类 Tab | 点击“运动/数码家电”等 | 切换顶部商品列表和比价结果 | 当前 UI 选中“运动”。
点击底部导航 | 点击首页/比价/探索/我的 | 跳转对应页面 | 比价 Tab 当前激活。
设计项 | 规范
面板形态 | 白色大圆角容器，顶部有灰色拖拽条，视觉上覆盖在页面下半部分。
面板状态 | collapsed：只展示普通比价列表；expanded：展示该商品的同款变体价格趋势。
卡片数量 | 首屏展示 4 张左右，列表可继续向下滚动。
卡片尺寸 | 横向大卡，左侧商品图、右侧价格曲线，下方来源与最低价。
返回逻辑 | 向下拖动滑条或点击空白区域可收起，回到比价页面。
数据来源 | compare.mock.json 中 selectedProduct 的 variants 数组。
区域 | 内容 | 样式要求
左侧商品图 | 同款不同颜色/套装商品图 | 白底商品图，主体居中，留白充足。
右侧曲线图 | 价格走势折线图 | 红色折线 + 浅粉面积 + 今天空心圆点。
Y 轴 | ¥240、¥320、¥400 等 | 灰色小字，位置固定。
X 轴 | 30天前、15天前、今天 | 灰色小字，减少信息噪音。
底部左侧 | 最低价来源：抖音商城 / 天猫 / 京东 / 拼多多 | 黑色正文，平台名加粗或保持普通。
底部右侧 | 最低价 ¥299 | “最低价”为黑色，价格红色加粗。
