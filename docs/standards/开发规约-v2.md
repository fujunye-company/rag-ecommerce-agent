> **最高优先级参考：`docs/background/REQS-竞赛核心需求.md`**  
> 本文档与 DEV-GUIDE.md 互补。冲突时以 REQS.md 为准。

---

## 一、目录规范

```
project/
├── apps/backend/        ← 提交时映射为 /server（符号链接）
├── apps/android/        ← 提交时映射为 /client（符号链接）
├── docs/                ← /docs
├── infrastructure/      ← docker/env
└── README.md
```

> **比赛提交要求**：目录使用 `/client`、`/server`、`/docs`。  
> 当前 `apps/` 前缀为工程习惯，提交时通过符号链接映射：  
> `ln -s apps/backend server && ln -s apps/android client`

---

## 二、命名规范

| 层 | 规范 |
|----|------|
| Python 文件 | snake_case |
| Kotlin 文件 | PascalCase |
| API 路由 | /api/resource, /api/resource/{id} |
| 数据库表 | 复数: products, sessions, messages, feedbacks |
| Git 分支 | feature/xxx → PR → squash merge main |

---

## 三、比赛红线（硬性 — 违反直接扣分）

| # | 红线 | 说明 |
|---|------|------|
| 1 | **严禁幻觉** | 不编造优惠券/功能/价格 |
| 2 | 原生客户端 | 不接受纯 Web/H5 |
| 3 | API Key 保密 | 不在代码中硬编码，使用 .env |
| 4 | **否定语义** | 正确处理"不要""除了""非""不含" |

## 四、功能要求（应有）

| # | 要求 | 说明 |
|---|------|------|
| 5 | 主动追问 | 信息不足时引导用户，不强行推荐 |
| 6 | 流式输出 | SSE 长连接，逐字渲染 |
| 7 | 商品卡片 | 对话中嵌入可交互卡片 |

---

## 四、代码规范

| 规范 | 要求 |
|------|------|
| 注释 | 核心逻辑（RAG链路、Prompt构造）必须注释 |
| 依赖 | requirements.txt 含版本号 |
| 错误处理 | API 层统一 try/except + 错误码 |
| 日志 | Python logging 分级（INFO/WARNING/ERROR） |
| 原子提交 | 一功能一 commit |

## 五、测试规范

| 级别 | 内容 |
|------|------|
| 单元测试 | 每个 service 模块 ≥5 条 pytest |
