# 核心要求

> 电商 AI 导购系统 — 项目核心需求描述
> 来源: 比赛题目

## 系统目标

基于 Android 原生框架、Python FastAPI 后端服务、向量数据库及大模型 OpenAPI 等技术栈，构建一个打通"意图理解-智能咨询-决策辅助"核心路径的电商 AI 导购系统。

## 关键能力

1. **深度意图理解** — 能够深度理解商品属性、用户意图并提供个性化决策支持的智能 Agent
2. **RAG 知识库** — 通过上传非结构化的商品详情与营销文档构建专属知识库，RAG 技术确保回复的专业性与准确性
3. **流式交互体验** — 利用原生技术栈打造媲美"豆包"的流式交互体验，支持商品卡片实时渲染
4. **多模态输入** — 支持文字/图片输入解析
5. **评测闭环** — 构建端到端的质量评测与反馈闭环，覆盖回答准确率、知识检索精度及多轮对话一致性

## 技术栈约束

- 客户端: Android 原生框架 (Kotlin + Jetpack Compose)
- 后端: Python FastAPI
- 数据库: PostgreSQL + pgvector
- 向量库: Qdrant / Chroma
- RAG: LlamaIndex / LangChain
- Agent: LangGraph
- LLM: DeepSeek / OpenAI-compatible API
