# 答辩 PPT 大纲 — Hermes 电商AI导购 Agent

> 比赛：第3届 AI 全栈挑战赛 | 版本：v1.0

## PPT 结构（建议 12-15 页，10 分钟）

### 第 1 页：封面
- **项目名称**: Hermes — 基于 RAG 的多模态电商智能导购 AI Agent
- **团队/作者**: Fu-junye
- **一句话定位**: "拍照就能找、说话就能买"的全场景 AI 导购

### 第 2 页：问题与痛点
- 传统电商搜索：关键词匹配，不理解用户意图
- 竞品对标：淘宝问问、豆包、Amazon Rufus
- 本项目的差异化：9 场景覆盖 + 多模态 + 否定语义 + 本地化

### 第 3 页：系统架构总览
- 架构图：Android (Compose) → FastAPI SSE → LangGraph Agent → Qdrant + PostgreSQL
- LLM 层：Doubao-Seed-2.0-lite (主) + DeepSeek (意图/槽位)
- Embedding：BGE-large-zh-v1.5 (1024-dim)

### 第 4 页：核心技术 1 — 意图识别 + 槽位填充
- 9 种意图分类（推荐/对比/详情/场景/反选/购物车/拍照找货/售后/闲聊）
- Fast LLM (DeepSeek ~0.5s) + 关键词回退
- 槽位提取：品类/价格/品牌/属性/场景 → missing_slots 驱动追问

### 第 5 页：核心技术 2 — 多轮对话与否定语义
- 对话历史注入意图分类器 + LLM prompt
- Clarify 反问 → 槽位合并继承
- 否定/反选条件提取 + Qdrant 过滤 + 文本兜底
- 演示：用户"推荐耳机" → "不要入耳式的" → 正确排除

### 第 6 页：核心技术 3 — RAG 检索管道
- Embedding → Qdrant 混合检索 → Reranker (bge-reranker-v2-m3)
- 分级回退：价格放宽 → 品类放宽 → 纯语义
- 兜底热榜推荐
- 检索质量监控：Precision@K 日志

### 第 7 页：核心技术 4 — 拍照找货 (VLM + RAG)
- 流程：Android Camera → Doubao Vision API → 商品描述 → 向量检索
- 降级：Doubao API 不可用时 → 关键词兜底
- 演示：拍照 → 秒级返回 3 件匹配商品

### 第 8 页：核心技术 5 — 流式 SSE 输出
- Progress → text_delta → product_cards → done 4 阶段
- 交错输出：文本 + 商品卡片同时流式推送
- 缓存版本校验：参数变更自动失效旧缓存
- TTFT < 1s (progress 事件即刻推送)

### 第 9 页：工程质量
- 全异步 FastAPI + async SQLAlchemy
- 全局异常中间件 + 请求耗时监控
- LLM 调用自动重试（指数退避：1s/2s/4s）
- 查询缓存 LRU (100条/5min TTL) + 购物车内存缓存
- LangGraph 状态图编排（低耦合节点）

### 第 10 页：评测结果
- 30 条快速评测：100% pass rate，平均延迟 < 1s
- 9 场景全覆盖
- M1-M10 里程碑全部完成

### 第 11 页：Android 客户端
- 6 页面：主页/对比/探索/知识/拍照/个人中心
- Jetpack Compose Material3
- 语音输入 (ASR) + 拍照 (Camera + Gallery)
- 多轮对话本地持久化

### 第 12 页：演示视频要点
- 列举 5-6 个演示场景
- 每个场景 30-60 秒

### 第 13 页：项目总结
- 9 场景 × 全栈覆盖 (Android + FastAPI + Qdrant + LLM)
- 工程成熟度：异常处理、重试、缓存、监控
- 比赛评分预估：基础功能 35% + 工程质量 25% + 效果 20% + 加分 20% ≈ 80%+
