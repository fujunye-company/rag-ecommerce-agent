## 设计文档

系统架构文档：[docs/ARCHITECTURE.md](https://github.com/fujunye-company/rag-ecommerce-agent/blob/main/docs/ARCHITECTURE.md)

技术栈与设计要点：

| 层 | 技术选型 | 设计理由 |
|----|----------|---------|
| Agent 编排 | LangGraph StateGraph (10 节点) | 有状态多步推理，支持分类→检索→生成→追问→对比→反选全链路 |
| 检索 | LlamaIndex + Qdrant (1024-dim) | 混合检索（语义+关键词）+ BGE-reranker-v2-m3 重排序 |
| 后端 | FastAPI SSE 流式 (8 事件类型) | 实时流式输出商品卡片/对比表/购物车，前端增量渲染 |
| 数据库 | PostgreSQL + pgvector | 结构化商品数据 + 向量双库同步 |
| LLM | Doubao-Seed-2.0-lite (DeepSeek 降级) | 比赛提供 Key，已通过非空验证 |
| Embedding | BGE-large-zh-v1.5 (本地) | 无需外网 API Key，离线可用 |
| 前端 | Kotlin + Jetpack Compose (72 .kt) | 原生 Android，非 H5/WebView |

## 说明文档

### 一键部署

```bash
git clone git@github.com:fujunye-company/rag-ecommerce-agent.git
cd rag-ecommerce-agent
./deploy.sh              # 启动全部服务（已运行时跳过）
./deploy.sh --build      # 启动 + 强制重建 backend 镜像
```

Docker Compose 自动启动 Postgres + Qdrant + FastAPI 后端，健康检查通过后即就绪。脚本会检测 3 个容器是否已 healthy，若已运行则跳过启动阶段。首次启动自动导入 288 条种子数据。

### 评委体验路径

1. **安装 APK**：`apps/android/app/build/outputs/apk/debug/app-debug.apk`（24.1MB）
2. **启动后端**：`docker compose up -d`（或 `cd apps/backend && uvicorn app.main:app --host 0.0.0.0 --port 8080`）
3. **配置地址**：APK 中填入 `http://<server-ip>:8080`
4. **体验 9 场景**：推荐（"推荐好用的手机"）→ 筛选（"3000 以内的耳机"）→ 追问 → 对比 → 反问 → 反选 → 场景化 → 购物车 → 拍照找货

### 演示数据

- 288 条商品，94 品类，覆盖手机/耳机/平板/手表/服装/美妆/食品/家居等
- 支持品牌筛选、价格筛选、属性筛选
- 购物车支持多商品、数量修改、删除

## 项目亮点/创新点

1. **品类感知 MMR 多样性采样**：传统 RAG 易被单一语义相关品类垄断，本方案在检索后引入品类感知的 MMR 采样，确保推荐结果覆盖多品类，避免"搜手机出 10 款手机"的同质化问题
2. **追问-对比-反选-场景化全链路闭环**：不仅支持推荐和筛选，还实现追问澄清（Clarify 节点）、多商品对比表格（Compare 节点）、否定反馈反选（"不要华为"自动排除并替换）、场景化导购（"想去海边度假"自动映射 5 品类推荐），覆盖真实导购 9 级场景
3. **SSE 8 事件类型流式输出**：自定义商品卡片事件（product_card）、对比事件（compare）、购物车事件（cart_update）、追问事件（clarify）等 8 种事件类型，Android 端增量解析渲染，优于纯文本流式响应
