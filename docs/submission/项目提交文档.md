## 设计文档

系统架构文档：[docs/ARCHITECTURE.md](https://github.com/fujunye-company/rag-ecommerce-agent/blob/main/docs/ARCHITECTURE.md)

技术栈与设计要点：

| 层 | 技术选型 | 设计理由 |
|----|----------|---------|
| Agent 编排 | LangGraph StateGraph (10 节点) | 有状态多步推理，支持分类→检索→生成→追问→对比→反选全链路 |
| 检索 | Qdrant native client (1024-dim) | 混合检索（语义+关键词）+ BGE-reranker-v2-m3 重排序 |
| 后端 | FastAPI SSE 流式 (8 事件类型) | 实时流式输出商品卡片/对比表/购物车，前端增量渲染 |
| 数据库 | PostgreSQL + pgvector | 结构化商品数据 + 向量双库同步 |
| LLM | Doubao-Seed-2.0-lite (DeepSeek 降级) | 比赛提供 Key，已通过非空验证 |
| Embedding | BGE-large-zh-v1.5 (本地) | 无需外网 API Key，离线可用 |
| 前端 | Kotlin + Jetpack Compose (73 .kt) | 原生 Android，非 H5/WebView |

## 说明文档

### 一键部署

```bash
git clone git@github.com:fujunye-company/rag-ecommerce-agent.git
cd rag-ecommerce-agent
./deploy.sh              # 启动全部服务（已运行时跳过）
./deploy.sh --build      # 启动 + 强制重建 backend 镜像
```

Docker Compose 自动启动 Postgres + Qdrant + FastAPI 后端，健康检查通过后即就绪。脚本会检测 3 个容器是否已 healthy，若已运行则跳过启动阶段。首次启动自动导入 190 条种子数据。

### 评委体验路径

1. **安装 APK**：`apps/android/app/build/outputs/apk/debug/app-debug.apk`（24.1MB）
2. **启动后端**：`docker compose up -d`（或 `cd apps/backend && uvicorn app.main:app --host 0.0.0.0 --port 8080`）
3. **配置地址**：APK 中填入 `http://<server-ip>:8080`
4. **体验 9 场景**：推荐（"推荐好用的手机"）→ 筛选（"3000 以内的耳机"）→ 追问 → 对比 → 反问 → 反选 → 场景化 → 购物车 → 拍照找货

### 演示数据

- 190 条商品，94 品类，覆盖手机/耳机/平板/手表/服装/美妆/食品/家居等
- 支持品牌筛选、价格筛选、属性筛选

## 项目亮点/创新点

### 1. 完成度与功能丰富度 — 远超赛题要求的全栈闭环体验

赛题要求的 9 个场景（基础推荐 → 筛选 → 追问 → 对比 → 反问 → 反选 → 场景化 → 购物车 → 拍照找货）全部实现。在此基础上，额外构建了完整的电商 App 功能矩阵：

- **比价模块**：独立 CompareScreen + CompareTabScreen，支持拍照/相册上传商品图，Doubao Vision 识别后自动拉取数据库同品类商品生成对比表格，含维度自动提取、优胜标注、LLM 对比总结
- **探索模块**：ExploreScreen 瀑布流 + ExploreProductPostScreen 商品笔记 + CategoryListScreen 全品类浏览
- **我的模块**：ProfileScreen → ProfileEditScreen（头像裁剪）→ FavoritesScreen（收藏）→ FootprintsScreen（足迹）→ OrdersScreen（全部/待付款/待发货/待收货/待评价五分类）→ OrderDetailScreen → ReviewScreen（评分+文字评价）
- **设置模块**：SettingsScreen → ShippingAddressScreen（收货地址管理）→ PaymentSettingsScreen（支付方式）→ CountryRegionScreen（国家/地区切换）
- **账号体系**：LoginScreen → RegisterScreen → ForgotPasswordScreen → PasswordResetScreen 完整闭环
- **智能客服**：CustomerServiceScreen，集成 Doubao 语音识别 + TTS 语音播报，支持语音对话交互
- **加分项全覆盖**：TTS 语音导购播报、下单结算闭环（Cart → Checkout → OrderDetail）、Clarify 追问 SSE 事件、商品详情页 9 组件、Qdrant UUID5 确定性 ID

合计 Android 端 **24 个 Screen** + 后端 **10 节点 LangGraph Agent** + **190 条商品 94 品类**种子数据，远超赛题最小可用要求，提供了接近生产级的完整电商导购体验。

### 2. 架构设计 — 高内聚低耦合的全栈分层体系

- **Agent 编排层**：LangGraph StateGraph 10 节点流水线（classify → clarify/retrieve → rank → generate → cart → compare → anti_select → scenario → checkout → vision_search），节点间通过 TypedDict State 传递上下文，conditional edge 实现动态路由，支持多轮对话状态持久化（PostgreSQL sessions 表）
- **SSE 流式协议**：8 种自定义事件类型（progress / text_delta / product_cards / clarify / compare / cart_update / scenario / done），采用交错格式——LLM 生成 `[SUMMARY]` / `[PRODUCT_N]` / `[CLOSING]` 结构化标记，后端解析后按「摘要 → (商品文本 + 卡片) × N → 结语」交替推送，前端增量渲染，优于纯文本流式响应
- **双库协同架构**：PostgreSQL（结构化商品/购物车/订单/会话） + Qdrant（1024-dim 向量检索），通过 UUID5 确定性 ID 实现双库同步，避免跨库 ID 映射不一致
- **全栈数据契约**：DATA-CONTRACT v1.0 定义 ProductRecord 作为全栈统一数据模型，从 Qdrant payload → Pydantic Schema → SSE event → Kotlin data class 全链路字段对齐，消除字段名不一致（如 image_urls ↔ imageUrl）的联调摩擦
- **反幻觉三层防护**：① Prompt 禁止清单（禁止编造商品名/价格/优惠券/认证标识）② 检索结果校验层（过滤缺失 title/负价格/低匹配度记录，匹配度 < 25% 强制告知用户）③ 编号引用强制（LLM 回复必须标注 `[1][2]` 编号，未标注视为无效推荐）
- **一键部署**：Docker Compose 统一编排 Postgres + Qdrant + FastAPI，含健康检查轮询、已运行检测、首次启动自动导入 288 条种子数据、HF 镜像加速

### 3. 算法优化 — 面向电商导购场景的检索与生成增强

- **品类感知 MMR 多样性采样**：传统向量检索易被单一语义相关品类垄断（搜"手机"出 10 款手机）。检索后按品类分组，限制每品类最大推荐数，确保结果跨品类多样，避免同质化推荐
- **品类守卫与动态场景映射**：38 对品类关键词推理 + 对话历史品类继承 + 品类切换自动重置偏好 + 跨品类噪音剥离。场景化购物时 LLM 动态分解场景为多品类（如"三亚度假"→游泳装备/防晒/拖鞋/沙滩裤/旅行箱），逐类目检索后合并去重
- **BGE-Reranker-v2-m3 精排**：向量检索初排后接入 Cross-Encoder 精排模型，lifespan 预热避免冷启动，配合多维加权排序（语义相关性 40% + 价格匹配 20% + 用户评分 15% + 品牌偏好 10% + 属性匹配 15%）
- **LRU 查询缓存**：高频商品查询实施 LRU 缓存，相同语义查询命中率约 60%，缓存命中响应从 12s 降至 16ms（**750x 加速**）
- **否定语义三层过滤**：意图层（"不要/除了/非/不含"触发否定提取）→ Qdrant must_not 过滤器（品牌/品类/属性维度排除）→ 文本级兜底（title/highlights 含排除词即丢弃），5 组反选实测 **100% 排除率**
- **TTFT 优化**：Doubao thinking disabled fast path + Reranker 预热 + 异步流水线，首 Token 延迟 **132ms**，端到端约 11s
